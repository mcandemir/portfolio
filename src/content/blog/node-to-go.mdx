## The Trigger

Our Node.js API was hitting a wall at ~5k concurrent connections. GC pauses were unpredictable, and the single-threaded event loop was becoming a bottleneck for CPU-bound work like JSON serialization of large payloads.

## Why Go?

- **Goroutines** — lightweight concurrency without callback spaghetti
- **Static typing** — catch entire classes of bugs at compile time
- **Single binary** — no `node_modules`, no runtime dependency
- **Performance** — consistent sub-millisecond GC pauses

## The Migration

We migrated service-by-service over 6 months, using an API gateway to route traffic. The key insight: don't rewrite everything at once. Start with the hottest path.

## Results

| Metric | Node.js | Go |
|--------|---------|-----|
| P99 Latency | 120ms | 12ms |
| Memory Usage | 512MB | 64MB |
| Throughput | 5k req/s | 50k req/s |
| Deploy Size | 200MB | 12MB |

The numbers speak for themselves.
