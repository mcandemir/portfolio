## Why a Terminal?

As backend engineers, we live in the terminal. It made sense to bring that aesthetic to the portfolio itself — not as a gimmick, but as a genuinely useful way to navigate content.

## The Architecture

The terminal is a React component rendered as an Astro island with `client:visible`. This means:

- Zero JavaScript shipped until the component scrolls into view
- The rest of the page remains fully static HTML/CSS
- Hydration happens lazily, keeping initial page load fast

## Command System

Commands are defined as a simple record mapping strings to output arrays. This keeps the logic declarative and easy to extend:

```typescript
const COMMANDS: Record<string, string[]> = {
  help: ["Available commands:", "  about  — Who am I?", "..."],
  about: ["Software Engineer", "..."],
};
```

Tab completion, command history (arrow keys), and clear are all supported for an authentic feel.

## Performance Impact

Even with this interactive island, the page maintains a 100/100 Lighthouse performance score. The key: ship zero JS for everything else.
